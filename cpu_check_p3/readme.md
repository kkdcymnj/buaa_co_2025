### Logisim CPU 对拍机使用说明

#### 写在前面

这个对拍机能够帮助您验证使用 **Logisim** 搭建的简易 CPU 是否能够正常工作。目前这个机器支持生成包含 `add`，`sub`，`ori`，`lw`，`sw`，`beq` 和 `lui` 指令的数据用于评测。
请注意：这台机器所生成的数据并没有很强，主要突出的是**随机性**。对于边界条件的测试，请自行构造相应数据。（11 月 2 日新增指令：`jr`，`jal`）

11 月 5 日更新：
如果你需要更为合理的数据，请下载 `cpu_checker_p4`，使用这个目录下的 `generator.py`。

#### 构成

本对拍机主要由以下几部分构成：

1. 数据生成器，生成结果将以 `.asm` 文件格式存储到 `sample` 目录下（`generator.py`）
2. 利用 MARS 导出汇编程序对应的机器码，存储到 `machine_code` 目录下（`runmars.py`）
3. 将机器码注入 ROM 进行评测，并比对输出，输出结果存储到 `results` 目录下（`runcirc.py`）

#### 使用方法

本对拍机的使用方法如下：

1. 在 `app` 目录下放置 `logisin-generic-2.7.1.jar` 和 `Mars.jar`，这两个包课程组均有提供。
2. 将需要进行对拍的 2 份电路文件放置到 `test_script` 目录下，分别命名为 `cpuA.circ` 和 `cpuB.circ`，且顶层模块名为 `main`；同时，这一目录下还会有测试电路 `testbench.circ`，请不要随意更改其配置。
3. 运行 `main.py`，生成数据并进行评测。运行 `main.py` 时：
- 选择是否需要重新生成数据
- 输入本次测试数据组数、每组数据指令条数（如果需要重新生成数据）。注意，这里的“指令条数”不包括对寄存器初始化的指令。
4. A、B 电路运行后的输出均储存在 `results` 目录下，分别命名为 `output_{i}A.txt` 和 `output_{i}B.txt`，其中 `i` 是评测数据编号。
对应的包含机器码的文件储存在 `machine_code` 目录下，命名为 `machine_test_case_{i}.txt`，包含汇编程序的文件储存在 `samples` 目录下，
命名为 `test_case_{i}.asm`。

#### 常见问题

1. 提示“文件不存在”：请仔细检查 `.circ` 是否已经放置到了 `test_script` 目录下，且已经按照要求重命名。
2. 提示“测试不能进行”：请仔细查看 `app` 目录下是否已经包含 `logisin-generic-2.7.1.jar`。
3. 提示“将电路导入时出错”：请仔细检查 `test_script` 目录下是否有测试电路 `testbench.circ`，以及原电路顶层模块名是否为 `main`。
4. 提示“请下载 MARS 并指定正确路径”：请仔细查看 `app` 目录下是否已经包含 `Mars.jar`。
