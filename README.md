# BUAA CO 课程仓库

## 作业代码部分（`cpu_code` 目录）

此处放出了本人 P3 ~ P7 部分的作业代码，均能够通过强测，正确性上没有明显问题。但是这些代码中可能会展现编写者恶劣的代码习惯，也有一些架构不够合理 / 冗余，请大家批判性地阅读。

无论如何也不要将这些代码直接复制变成你的作业！CO 课程的查重会覆盖**历届**提交内容，抄袭我的代码会导致作业 0 分，也不利于你理解 CPU 架构，从而更不利于完成课上新增指令的考核。

## 评测机部分（`cpu_check_px` 目录）

### 说明

这些评测机能够生成测试数据以评测自行搭建的 CPU。不过本人不能保证数据具有足够的强度，所以也请自行构造测试数据来检验。

P3 的评测机只能对拍，P4 及之后的评测机可实现“自测”。有兴趣的同学可以尝试解析 Logisim 输出的二进制文件来实现 P3 的“自测”。

每一个 P 的评测机的构造大体相同，但细节上又有所不同。使用前请仔细阅读对应目录下的 `readme.md` 文件。如果在使用中出现了任何难以解决的问题，可以通过邮件 `24371491@buaa.edu.cn` 联系。

### 鸣谢

感谢 Tony 学长等人开发的魔改版 Mars！它大大减少了我们实现自测的工作量。

### 局限性

1. P3 的评测机数据较弱，且并不总是合法，容易出现死循环的状况。建议用 P4 及以后评测机目录下的 `generator.py` 文件来替换 P3 目录下的同名文件。

2. 由于 MARS 本身的局限性，P7 仅能实现对异常的检查，中断部分的测试请与同学对拍。此外，在 MARS 中写地址 0x2fff 会被判定为异常，但在课程作业中这并不是异常，请辩证看待 MARS 运行结果。
